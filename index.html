What you'll need

No matter how many tools you see, they all do one of five things:

01

The Editor / IDE

Where you write and read code. An IDE (Integrated Development Environment) is a text editor with superpowers — it understands your code, highlights errors, helps you navigate, and runs your programs.

VS Code · PyCharm · Cursor
Learn about IDEs →
02

The AI Assistant

A model that helps you write, explain, debug, and refactor code. Can live inside your IDE or as a separate chat interface. Transforms what a solo developer can accomplish in a day.

Claude · GitHub Copilot · ChatGPT · JetBrains AI
Learn about AI tools →
03

Python Environment (venv)

A self-contained Python installation for each project. Keeps your projects isolated so their dependencies don't conflict — this is about managing packages and versions, not about AI. Every professional Python project has one.

venv · conda · uv
Learn about environments →
04

Version Control

A system that tracks every change to your code, lets you go back in time, and enables multiple people to work on the same project without overwriting each other.

Git · GitHub · GitLab · Bitbucket
Learn about Git & GitHub →
05

Package Management

How you install, update, and manage the libraries your code depends on. Works alongside your virtual environment to keep everything reproducible and shareable.

pip · conda · uv
Learn about packages →

Five levels of AI integration

Five levels of AI integration

All of these levels use the same underlying AI model. What changes is how much of your environment the AI can see and act on — and how much you have to do manually in between. Knowing where you are shows you what to learn next. Read more here.

Level 0–1

Browser chat + copy-paste

You're using ChatGPT or Claude in a browser tab. You describe a problem, get code back, paste it into your editor, run it, hit an error, copy the error back up, and repeat. Or you have Copilot in your IDE doing inline suggestions — helpful, but still basically a chat loop. The core limitation: the AI has no access to your project. It can't see your files, your errors in context, or how your code is structured. Every exchange starts from scratch.

The Get Started page walks you through a complete setup in VS Code and PyCharm — including AI from day one.
Level 2

AI in your editor

The AI can now read and edit your actual project files. You describe what you want; it proposes changes across multiple files at once. You review the diffs and decide what to keep. The AI isn't just suggesting — it's doing. The shift here isn't about a specific tool, it's a pattern: the AI has access to your codebase, and you're in the role of reviewer rather than typist. This works in VS Code with GitHub Copilot agent mode, and in other editors that support in-context AI.

→ Level 2: AI in Your Editor
Level 3

Dedicated coding agents

The AI moves out of the editor and into your terminal, where it has access to your full computing environment. This means it can run code and see the output, not just read files. It can execute scripts, install packages, use system tools, read and write anywhere on your filesystem, and iterate on its own until a task is complete. You describe an outcome; it figures out the steps. The IDE is still useful for reviewing what the agent produced, but it's no longer required to run the task. Claude Code is the primary example covered here; OpenAI Codex CLI and Gemini CLI follow the same pattern.

→ Level 3+
Level 4

Connected agents

You extend the agent with live connections to the outside world via MCP (Model Context Protocol) — a standard that lets the agent pull in information from beyond your computer: current library documentation, web search, databases, and other services. Without this, an agent is limited to what it already knows, which can lag months behind. With it, the agent can look up the exact API it needs right now, query your database while it codes, or search the web mid-task. Superpowers, Context7, and Sequential Thinking from the Level 3+ page are Level 4 tools layered on top of a Level 3 setup.

Level 5

Fully autonomous runs

You give the agent a goal, set it running in a container, and come back an hour later. Real, but requires significant setup, careful scoping, and thorough review of the result.

All levels use the same underlying AI model. The difference is how much structure, access, and autonomy you give it.
