Five categories. The structure behind every Python setup.

Every tool fits into one of these roles.

No matter how many tools you see, they all do one of five things:

01

The Editor / IDE

Where you write and read code. An IDE understands your code, highlights errors, helps you navigate, and runs your programs.

VS Code · PyCharm · Cursor

02

The AI Assistant

A model that helps you write, explain, debug, and refactor code. Can live inside your IDE or as a separate chat interface.

Claude · GitHub Copilot · ChatGPT · JetBrains AI

03

The Environment

A self-contained Python installation for each project. Keeps your projects isolated so their dependencies don't conflict.

Version Control

A system that tracks every change to your code, lets you go back in time, and enables multiple people to work together.

Stages of AI-assisted coding

What level are you at?

AI can only help at a depth you can follow. Knowing where you are shows you what to learn next. Framework from this guide. This site covers Levels 0–3.

Level 0–1

Browser chat + basic IDE

You use ChatGPT or Claude in a browser tab, copy code into your editor, run it, copy errors back up, and repeat. Or you have Copilot in VS Code giving inline suggestions. This is where most students start.

The Get Started page walks you through a complete Level 1 workflow in VS Code and PyCharm — including AI from day one.
Level 2

Agentic IDE

You have an AI-first editor like Cursor. You tell the AI what you want, and it edits files, reads your codebase, and works across multiple files at once. The AI isn't just suggesting; it's doing. You review diffs and decide what to keep.

Ready for Level 2? See the Cursor guide →
Level 3

Dedicated coding agents

Tools like Claude Code run in your terminal with access to your full computing environment — any file, any script, any tool. They can think about a task, write code, run it, see the output, and iterate autonomously. The IDE becomes optional.

Ready for Level 3? See Claude Advanced →
Level 4

Connected agents (MCP)

You extend your agent via MCP (Model Context Protocol) — connecting it to live documentation, databases, and external APIs. The agent can now pull current library docs, query your data, or search the web while it codes.

Level 5

Fully autonomous runs

You give the agent a goal, set it running in a container, and come back an hour later. Real, but requires significant setup, careful scoping, and thorough review of the result.

All levels use the same underlying AI model. The difference is how much structure, access, and autonomy you give it.
