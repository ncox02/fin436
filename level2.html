04
Write Finish Starter Code

Create compound.py and paste this starter code:

Fallback Гўв‚¬вЂќ send this in Copilot Chat:

Note on the formula: the function should return the final balance Гўв‚¬вЂќ the full amount including principal Гўв‚¬вЂќ using A = P(1 + r/n)^(nt). If AI returns only the interest earned, ask it to revise: "Return the final balance, not just the interest earned."

Writing intention before implementation produces more accurate completions than typing code directly. The comment block is the specification.
05

Prompt Engineering

Prompt Engineering

Why Exercises 1 & 2 worked Гўв‚¬вЂќ and how it scales

You just built a small spec and a scaffold. In serious AI-assisted projects, this pattern has a name and a folder structure that reflects it directly:

  • spec/ what the thing is: goals, constraints, non-goals
  • design/ how it is structured: architecture, data model, component map
  • context/ what AI needs to know each session: current state, decisions made, open questions
  • tasks/ discrete, completable units of work

When you open a new chat with a vague description, AI is guessing at your intent. When you hand it a precise spec, it is executing against known constraints. Those are completely different quality levels Гўв‚¬вЂќ and the difference compounds over time.

This is why you are not constantly backpedaling: the work is in the documents, not the chat history. The habit starts at Exercise 1.

On Managing Scope

Four principles that separate productive sessions from overwhelming ones

1

Give context before the ask

Describe what the project does and what role this code plays before making the request. Without context, AI fills gaps with assumptions.

ГўЕ“вЂў "Write a function that validates input."
ГўЕ“вЂњ "I'm building a compound interest calculator. Write a function that validates that principal, rate, and years are all positive numbers and raises a ValueError with a descriptive message for each invalid case."
2

Constrain the scope explicitly

Without limits, AI defaults to comprehensive Гўв‚¬вЂќ more than you need, more than you can review. Explicit constraints keep the output manageable.

ГўЕ“вЂў "Add error handling."
ГўЕ“вЂњ "Add a try/except block to the calculate_compound_interest call in main.py only. Catch ValueError and print the message. Don't touch anything else."
3

Ask for a plan before you ask for code

For anything larger than a single function, ask AI to describe its approach first. Exercise 1 does this explicitly Гўв‚¬вЂќ it is the right habit for any new project.

ГўЕ“вЂњ "Before writing any code, describe how you'd structure a function that compares compounding frequencies. What edge cases would you handle?"
4

Tell it your level and what you want to learn

"I'm new to pandas" produces simpler methods and more explanation. "Explain your reasoning alongside the code" gets you something to learn from, not just copy.

ГўЕ“вЂњ "I'm learning pandas. Write this using basic operations and add a comment on each line explaining what it does."

Editor & Context

Your editor, your choice

VS Code with GitHub Copilot and PyCharm with JetBrains AI Assistant are the most common starting points. Cursor is a fork of VS Code rebuilt with in-context, inline AI editing as its core purpose rather than a plugin Гўв‚¬вЂќ popular and worth a look. Zed, Windsurf, and others follow the same idea. The capability matters; the specific tool is secondary.

How file context works

When you open a file and open the Copilot Chat sidebar, the AI has already read your open file. Type #filename.py in your chat message to pull a specific file into context explicitly. AI sees across files simultaneously Гўв‚¬вЂќ something browser chat cannot do without copy-pasting each one. File context becomes meaningful starting at Exercise 5; the earlier exercises work from plain-English descriptions alone.

What's next

Level 3 Гўв‚¬вЂќ AI in your terminal

Level 2 keeps AI inside the editor. It reads your files and proposes changes Гўв‚¬вЂќ but it cannot run anything. You still execute code in the terminal and carry results back into the chat manually.

Level 3 removes that constraint. A terminal agent like Claude Code has access to your full computing environment: it executes scripts, reads the output, installs packages, modifies files, and iterates until a task is complete. Exercise 9 above was the first preview of that boundary. Level 3 is what happens when you remove it entirely.
