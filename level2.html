Level 2: AI in Your Editor

At Level 1, AI lives in a browser tab — you copy code out, paste errors in, and manually carry context between two windows. Level 2 changes the fundamental setup: the AI moves directly into your codebase, where it can read your files, write code inline, and hand you a diff to review. You stop being a messenger and start being a reviewer. This page shows you exactly how to set that up — in VS Code and PyCharm — and how to use it well.

The In-Editor Shift

This is not a faster version of copy-paste

The old browser chat approach isn't slow because the AI is worse — it's slow because you're doing all the carrying. Switching windows, reconstructing context from scratch, manually applying suggestions, and re-copying every error. Level 2 eliminates that overhead entirely.

The Old Way — Browser Chats

The copy-paste loop

  • Tab-switch from editor to browser and back, dozens of times per session
  • Manually copy code fragments into chat, losing surrounding context
  • Paste AI output back into the editor, often requiring manual cleanup
  • Copy error messages out of the terminal, paste them into the chat
  • Each new chat starts from scratch — no memory of your project
  • AI can't see how your files relate to each other
  • Suggestions frequently conflict with code it hasn't seen
The In-Editor Way — Level 2

Chat in context, edit inline

  • Chat panel lives inside your IDE — no window switching
  • AI reads your actual project files before responding
  • Changes are proposed inline, directly in the file
  • Diffs are shown side-by-side — you review and accept or reject
  • Highlight any function and ask for targeted edits in place
  • AI sees cross-file relationships and can reason across your whole project
  • Suggestions are grounded in your real code structure
🕐
What stays the same

Git, project architecture, virtual environments, and every core software engineering principle you've already learned remain unchanged. The five-step sequence — Organize → Write → Test → Share → Deploy — doesn't change either. Level 2 is purely about how you interact with AI at each step, not a different kind of project or a different workflow structure.

⚡
A note on Cursor

This guide uses VS Code and PyCharm because they're the most widely used starting points. But Cursor — an AI-first fork of VS Code built entirely around this kind of in-context, in-editor AI interaction — is gaining broad popularity among developers for exactly the reasons this page describes. If you're already comfortable in VS Code and want an editor that takes the Level 2 approach even further by making it native rather than plugin-based, Cursor is worth a look. Your extensions, keybindings, and settings transfer over in one click.

Environment Setup

Getting AI inside your editor

Choose your editor below and follow the steps. Once setup is complete, the AI has read access to your project files and you have an inline chat panel and inline edit capability.

Your editor:
1

Install the GitHub Copilot extension

Open VS Code. Go to the Extensions panel (Ctrl+Shift+X / Cmd+Shift+X). Search for GitHub Copilot and install it. Then search for GitHub Copilot Chat and install that too — the chat panel is separate from the inline completions extension.

2

Sign in with your GitHub account

After installation, VS Code will prompt you to sign in to GitHub. Click Sign in to GitHub and complete the browser OAuth flow. Your GitHub account is now linked to Copilot inside the editor.

3

Verify it's working

Open any .py file. Start typing a comment or a function signature. You should see gray ghost text appear — that's Copilot's inline completion. Press Tab to accept it. Then open the Chat panel via the sidebar icon (speech bubble) or Ctrl+Alt+I and type a question about your code.

4

Understand the pricing

GitHub Copilot's Free tier includes 2,000 inline completions and 50 chat messages per month — enough to evaluate it properly. The Pro plan is approximately $10/month and removes those limits.

Free — 2,000 completions / 50 chats per month Pro — ~$10/month, unlimited

PyCharm Professional — enable JetBrains AI Assistant

If you have PyCharm Professional, JetBrains AI Assistant is available under Settings → Plugins → Marketplace. Search for AI Assistant, install it, and restart. Sign in with your JetBrains account when prompted. The chat panel appears on the right sidebar.

The AI Assistant reads your open files and project context automatically, and supports inline code generation, chat, and code explanations out of the box.

Included with PyCharm Professional — ~$25/month

Inline completions will appear as you type. The Copilot Chat panel is available under the Tools menu.

Hands-On Walkthrough

Earnings surprise analysis, built in-editor

You'll build a script that reads a stock earnings CSV with columns ticker, quarter, estimated_eps, and actual_eps, calculates earnings surprise percentages, flags significant misses, and generates a plain-text summary. Every step is done using the in-editor AI tools you just set up.

Your OS:
Your editor:
1
Organize
Project initialization

Open a terminal and run the following to create a project folder, set up a virtual environment, and install your dependencies.

# Windows — Command Prompt or PowerShell mkdir earnings-analysis cd earnings-analysis python -m venv venv venv\Scripts\activate pip install pandas pytest
# macOS — Terminal mkdir earnings-analysis cd earnings-analysis python3 -m venv venv source venv/bin/activate pip install pandas pytest

Now open the project in your editor:

# VS Code — run from inside the project folder code .

In PyCharm, go to File → Open and select the earnings-analysis folder. When prompted, configure the Python interpreter to use the venv you just created.

2
Organize
AI-assisted project planning

Before writing a single line of code, use the AI chat panel to map out a sensible folder structure for this project. This is one of the biggest advantages of in-editor AI — you get an architecture proposal grounded in your actual task before you've committed to any structure.

Open the Copilot Chat panel (Ctrl+Alt+I or the speech-bubble icon in the sidebar). Send this prompt:

Open the AI Assistant chat panel (right sidebar or Tools → AI Actions → Open AI Chat). Send this prompt:

I'm building a Python project called earnings-analysis. It will read a CSV file with columns: ticker, quarter, estimated_eps, actual_eps. The script should calculate earnings surprise percentage for each row, flag rows where the surprise exceeds ±5%, and produce a plain-text summary. Please propose a folder structure and list of files for this project before I create anything. Include a src/ module, a tests/ folder, and a data/ folder for the CSV. Keep it simple — no frameworks, just clean Python modules.

Review the AI's proposal before creating any files. Does the module breakdown make sense? Is there anything you'd reorganize? Adjust the plan in the chat until it matches your intent, then create the folders and empty files manually or ask the AI to list the exact commands to do so.

3
Write
Core logic via inline completions

Open src/analysis.py. The most efficient way to write the core function is to type a detailed descriptive comment above the function signature, then let the AI complete it. The comment is the specification — be precise and the completion will be accurate.

# Calculate earnings surprise as a percentage. # surprise_pct = ((actual_eps - estimated_eps) / abs(estimated_eps)) * 100 # Returns a new DataFrame with an added 'surprise_pct' column. # Handles the case where estimated_eps is zero by returning NaN for that row. def calculate_surprise(df):

After typing the function signature, pause. Ghost text will appear after a moment, suggesting the entire function body. Read it. If it looks correct, press Tab to accept. If only part of it is right, accept with Tab and then edit — it's faster than writing from scratch.

Use the same approach for the flag_significant_misses function: write a precise comment describing the threshold logic, the column name to add, and the boolean values expected, then let the completion fill in the body.

4
Write
Refinement via inline edit

Now suppose you want to add a generate_summary function, but after writing it, you realize it needs to handle an edge case: what if the input DataFrame is empty? Select the existing function body with your cursor, then trigger inline edit:

Press Cmd+I (Mac) or Ctrl+I (Windows) with the code selected. A small inline input appears. Type your request:

Select the function, then press Ctrl+\ to trigger inline AI edit, or right-click and choose AI Actions → Suggest Refactoring. Type your request in the prompt field that appears:

Add a guard at the top of this function that returns the string "No earnings data available." if the DataFrame is empty. Don't change anything else about the function.

The editor will present a side-by-side diff showing exactly what the AI proposes to change. This is the critical moment: read the diff carefully. Confirm the guard was added in the right place, that nothing else was modified, and that the indentation and style match the rest of the file. Click Accept only when you're satisfied. If it's wrong, click Discard and refine your request.

5
Test
AI-generated test suite

Open src/analysis.py. Select all three functions — calculate_surprise, flag_significant_misses, and generate_summary. With the code selected, open the chat panel and send this prompt:

I've selected three functions from my analysis module. Please generate a comprehensive pytest test file for them. Include: - Normal cases with typical data - Edge cases: empty DataFrame, single row, all-zero estimated EPS, very large surprise values, negative EPS values - One test that verifies the summary output contains the word "surprise" Save the output as tests/test_analysis.py and use only pytest and pandas — no other libraries.

Save the generated file to tests/test_analysis.py, then run the suite from the terminal:

venv\Scripts\activate pytest tests/ -v
source venv/bin/activate pytest tests/ -v

If any tests fail, copy the failure output directly from the terminal and paste it into the chat panel:

Here are the pytest failures: [paste your terminal output here] Please fix the function that's causing these failures. Show me the corrected code.

Review the proposed fix in the diff, understand what was wrong, then accept and re-run pytest until all tests pass.

The Level 2 Blueprint

The full workflow, powered by an in-editor AI

Here's how the four core phases of every Python project change when you're working at Level 2. The phases are identical — only the execution changes.

Organize

📦 Plan first, file later

  • Use the chat panel to describe your project before creating any files
  • Ask for a folder structure and module breakdown
  • Review the AI's proposal critically — adjust before committing
  • Let AI generate the scaffolding commands once you've agreed on the structure
Write

✍ Comment → Complete → Refine

  • Write precise comments above function signatures — the comment is the spec
  • Let inline completions fill in the body; press Tab to accept
  • For targeted changes, select the code and use inline edit (Cmd+I / Ctrl+I / Ctrl+\)
  • Read every proposed diff before clicking Accept — this is the quality gate
Test

🧪 Generate → Run → Feed back

  • Select your core functions and ask the AI to generate exhaustive pytest cases
  • Include edge cases in your prompt — don't rely on the AI to guess them
  • Run pytest in the terminal; paste failures directly into the chat
  • Iterate on failures in the chat until all tests pass
Share

🔁 Review diffs before you commit

  • Git workflow is unchanged — git add, git commit, git push
  • Before writing a commit message, ask the AI to audit your staged changes
  • Prompt: "Review my staged diff for bugs, style issues, or anything I should fix before committing"
  • Ask the AI to suggest a concise, descriptive commit message once you're satisfied

What's Next

The Level 3 leap

Level 2 keeps AI inside the editor — a powerful upgrade, but still tethered to a UI. Level 3 removes that constraint entirely.

From editor plugin to environment-aware terminal agent

At Level 2, the AI reads your files and proposes changes — but it can't actually run anything. You still carry outputs from the terminal back into the chat manually. Level 3 changes that. A terminal agent like Claude Code lives in your command line and has access to your full computing environment: it can execute scripts, read the output, install packages, write and modify files anywhere, and iterate on its own until a task is complete.

The key difference: at Level 2 you describe a change and review it. At Level 3, you describe an outcome and the agent figures out the execution steps — running code, reading errors, fixing them, and continuing — without you manually carrying outputs between a terminal and a chat window. The IDE is still useful for reviewing what the agent produced, but it's no longer required to drive the task.
