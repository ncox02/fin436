AI in Your Editor

AI moves inside your editor — reads your open files, proposes changes inline, and shows you a diff before touching anything.

The shift

Not a faster version of copy-paste

The old way — browser chat

The copy-paste loop

  • Switch tabs between editor and browser dozens of times per session
  • Copy code into chat, losing surrounding context
  • Paste output back, often needing manual cleanup
  • Copy terminal errors out, paste them back in
  • Every new chat starts from scratch
  • Suggestions conflict with code the AI hasn't seen
Level 2 — in-editor AI

AI where your code lives

  • Chat panel lives inside your IDE — no window switching
  • AI reads your open file before you ask anything
  • Changes proposed inline, in the file you're editing
  • Diff shown before anything is touched — you decide
  • Select any block and ask for targeted edits in place
  • Suggestions grounded in your real code

10 Things to Try in Your Editor

Each exercise is self-contained — do them in any order. Exercises 4–8 share the same code block, but none depend on completing the others first.

1
Organize
Propose a project structure
See AI reason about structure before a single file exists

Open the chat panel with no files open. Send the prompt, read what comes back, push back if the structure doesn't make sense. When satisfied, ask it to list the terminal commands to create the folders and files.

I'm building a Python project that reads a folder of CSV files, cleans the data, and produces a summary report. Keep it simple — no frameworks, just clean Python modules. Suggest a folder structure and list each file with a one-line description of its purpose. Don't write any code yet.

Key moves: plain-English description, explicit constraint ("no frameworks"), "don't write any code yet" keeps you in control of pacing. Swap in any project idea you already have.

2
Write
Generate code from a description
A complete function from nothing but plain English

Open a new file called temperature.py, open the chat panel, and send the prompt. Read what comes back before accepting — does it match? Anything to adjust?

Write a Python function called convert_temperatures that takes a list of temperatures in Celsius and returns a new list converted to Fahrenheit. Handle the case where the input list is empty. Add a short comment explaining the formula. No external libraries.

No starter code, no file context — just a description. Specific constraints produce tighter results. Vague prompts get vague code.

3
Write
Finish code from a starter
Write the intention, let AI handle the implementation

Create summarize.py and paste the code below. Place your cursor after the last comment line and press Enter. Ghost text should suggest the function body — read it before pressing Tab. If nothing appears, use the fallback prompt instead.

import pandas as pd def summarize_returns(filepath: str, ticker: str) -> dict: # Load a CSV with columns: date, ticker, close_price. # Filter rows for the given ticker. # Compute mean close price, max close, and min close. # Return results as a dictionary with keys: mean, max, min.

Fallback — send this in the chat panel:

Complete the function body for summarize_returns based on the comments inside it. Use only pandas. Return the result as a plain dictionary.

The comment is the specification. More precise comment = more accurate completion.

4
Understand
Explain a block of code
AI explaining code is often more useful than AI writing it

Create pipeline.py and paste the function below. It works, but has inefficiencies and a subtle bug. Select the entire function, open the chat panel, send the prompt. Exercises 5–8 all use this file.

import pandas as pd import time def process_sales(df: pd.DataFrame, year: int) -> pd.DataFrame: results = [] for i in range(len(df)): row = df.iloc[i] if row["year"] == year: margin = (row["revenue"] - row["cost"]) / row["revenue"] results.append({ "product": row["product"], "revenue": row["revenue"], "margin": margin }) summary = pd.DataFrame(results) summary = summary.sort_values("margin") total = 0 for i in range(len(summary)): total = total + summary.iloc[i]["revenue"] summary["revenue_share"] = summary["revenue"] / total return summary

Explain this function step by step. Note any Python or pandas concepts worth understanding, and flag anything inefficient or risky.

Works on any code you didn't write — inherited projects, library internals, online examples. Explain before you change.

5
Debug
Spot a bug
A second pair of eyes that has read every Python bug report ever written

The process_sales function has a subtle bug. Select it in pipeline.py, open the chat panel, and send the prompt. See if AI finds it before you do.

Review this function for correctness. Is there any input that would cause a wrong result or raise an error? Be specific about where the problem is and why it occurs.

The bug: when row["revenue"] is zero, the margin calculation divides by zero — producing inf or a ZeroDivisionError depending on data type. It only surfaces on real data.

AI as a quality check, not a builder. It catches structural mistakes like this reliably.

6
Refactor
Improve performance and measure it
Give AI slow code, get fast code back — then prove it with a timer

process_sales iterates row by row — the slowest way to process data in pandas. Generate test data, ask AI to rewrite it using vectorized operations, then run the timing block to compare both versions in your terminal.

Step 1 — Create the data. Paste into a new file and run once:

import pandas as pd import numpy as np df = pd.DataFrame({ "product": np.random.choice(["Widget", "Gadget", "Doohickey"], 50000), "year": np.random.choice([2022, 2023, 2024], 50000), "revenue": np.random.uniform(100, 10000, 50000).round(2), "cost": np.random.uniform(50, 8000, 50000).round(2), }) df.to_csv("sales_data.csv", index=False) print("sales_data.csv created — 50,000 rows")

Step 2 — Refactor. Select process_sales in pipeline.py and send:

Refactor this function to use vectorized pandas operations instead of iterating row by row. Fix the division-by-zero bug in the margin calculation. Name the new function process_sales_fast. Keep the output identical — same columns, same structure.

Step 3 — Measure. Add this to the bottom of pipeline.py and run the file:

if __name__ == "__main__": df = pd.read_csv("sales_data.csv") start = time.perf_counter() process_sales(df.copy(), 2023) t_original = time.perf_counter() - start start = time.perf_counter() process_sales_fast(df.copy(), 2023) t_fast = time.perf_counter() - start print(f"Original: {t_original:.4f}s") print(f"Refactored: {t_fast:.4f}s") print(f"Speedup: {t_original / t_fast:.1f}x faster")

Expect 10–50x faster on 50,000 rows. Vectorized operations work on entire columns at once rather than one row at a time.

7
Annotate
Add type hints
Make your function's contract explicit in seconds

Select process_sales_fast from Exercise 6. Trigger inline edit and send the prompt. Review the diff before accepting.

Inline edit: Cmd+I Mac  /  Ctrl+I Windows

Inline edit: Ctrl+\  —  or right-click → AI Actions

Add complete Python type hints to all parameters and the return value. Use specific types — not just Any or object.

Type hints don't change runtime behavior — they make mismatches visible before you run anything.

8
Document
Generate a docstring
AI writes documentation developers find tedious

A docstring lives inside a function right below the def line, wrapped in triple quotes. Run help(process_sales_fast) in your terminal and Python prints it.

Select the refactored, type-hinted function from Exercise 7 and send the prompt.

Write a Google-style docstring for this function. Include a one-line summary, each parameter with its type, and the return value. Keep it concise.

NumPy and Sphinx are common style alternatives. Specify your team's standard in the prompt and AI will follow it.

9
Test
Generate pytest cases
Ask AI to write tests for code it has already read

Open pipeline.py. Select process_sales_fast and send the prompt.

Write three pytest functions for process_sales_fast — one normal case, one where no rows match the given year, and one where revenue is zero for some rows. Use only pytest and pandas. No fixtures.

Save the output to test_pipeline.py. Read each test before running it. Then run pytest test_pipeline.py -v.

AI-generated tests are a starting point. They occasionally test the wrong thing — read them before trusting them.

Working well with AI

Learning to Prompt Well

AI matches whatever scope you give it. Vague or enormous gets vague or enormous back. The principles below separate a productive session from an overwhelming one.

1

Give context before the ask

Describe what the project does and what role this code plays before making a request. Without context, AI fills gaps with assumptions that are often wrong in ways that are hard to spot.

✕ "Write a function that cleans data."
✓ "I'm building a sales reporting script. I have a DataFrame with revenue and cost columns that sometimes contain nulls or negative values. Write a function that drops rows where either column is null or negative."
2

Constrain the scope explicitly

Without limits, AI defaults to comprehensive — more than you need, more than you can review. "Under 20 lines," "no external libraries," "just the function, no explanation" keep the output reviewable.

✕ "Add error handling to my project."
✓ "Add a try/except block to the load_csv function only. Catch FileNotFoundError and print a clear message. Don't touch anything else."
3

Ask for a plan before you ask for code

For anything larger than a single function, ask AI to describe its approach first. It lets you redirect before you're buried in output you don't want.

✓ "Before writing any code, describe how you'd structure a function that does X. What edge cases would you handle?"
4

Tell it your level and what you want to learn

"I'm new to pandas" produces simpler methods and more explanation. "Explain your reasoning alongside the code" gets you something to learn from, not just copy.

✓ "I'm learning pandas. Write this using basic operations and add a comment on each line explaining what it does."
Example prompt — all four principles combined
// context I'm building a Python script that reads monthly sales CSVs and produces a summary report. Each CSV has columns: product, region, revenue, and cost. I need a function that loads one CSV, drops rows where revenue or cost is missing, and returns a cleaned DataFrame. // constraints Keep it under 20 lines. Use only pandas and the standard library. No classes — just a single function. // plan before code Before writing the function, describe how you'd handle the case where the file doesn't exist or is empty. Then write the code. // level I'm comfortable with Python basics but newer to pandas. Add a short comment on any pandas method that isn't obvious.

When you feel lost or overwhelmed

A response that's too long or too complex is feedback about the prompt, not your ability to code.

The response is more complex than you expected

Push back in the same chat: "That's more than I need — simplify it. Just the core logic, no extra abstractions." Simplification is a normal follow-up and AI handles it well.

You've lost the thread mid-conversation

Long conversations accumulate context that works against you. Start fresh: "Ignore the previous approach. Let's start over — I need something simpler."

It looks right but might not be

Confidence is not correctness. AI can produce fluent code with a subtle logic error or wrong assumption. Running the code is always the check. When something fails, paste the exact error back into the chat.

Good to know

Editor & Context

Your editor, your choice

VS Code with GitHub Copilot and PyCharm with JetBrains AI Assistant are the most common starting points. Cursor is a fork of VS Code rebuilt with in-context, inline AI editing as its core purpose rather than a plugin — popular and worth a look. Zed, Windsurf, and others follow the same idea. The capability matters; the specific tool is secondary.

How file context works

When you open a file and open the chat panel, the AI has already read it. Reference files explicitly in VS Code Copilot Chat by typing #filename.py in your message. In PyCharm AI Assistant, the open file is included automatically — drag additional files into the chat to add more.

What's next

Level 3 — AI in your terminal

In-editor AI reads your files and proposes changes — but it cannot run anything. You still execute code in the terminal and carry results back into the chat manually.

Level 3 removes that constraint. A terminal agent like Claude Code has access to your full computing environment: it executes scripts, reads the output, installs packages, modifies files, and iterates until a task is complete. You describe an outcome; the agent handles the execution loop.
