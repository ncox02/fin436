Cursor

At Level 1, you give AI line-by-line instructions. At Level 2, you give it goals. Cursor is a fork of VS Code rebuilt around the idea that AI should work across your entire codebase — reading files, proposing changes, and executing tasks — while you stay in the driver's seat reviewing diffs.

What changes at Level 2

At Level 1 (VS Code or PyCharm), AI is a tool you reach for. You write a comment and accept a suggestion. You paste an error into the chat and get an explanation. You select a function and ask for a docstring. These are valuable, but you're still directing every individual action.

Level 2 shifts the model. Instead of directing line-by-line, you describe an outcome and let the AI figure out how to achieve it across however many files it takes. You review what it did and decide what to keep.

Level 1 interaction

“Write a function that loads a CSV and handles missing values.”
AI writes one function. You review it.

Level 2 interaction

“Add proper error handling to the entire data pipeline. Make sure every file operation is wrapped and failures are logged.”
AI reads all relevant files, proposes changes across multiple files, shows diffs. You review and approve.

What stays the same

The five-step workflow (Organize → Write → Test → Share → Deploy) doesn't change. The project structure doesn't change. Git and GitHub don't change. Level 2 is about how you interact with AI — not a different kind of project.

💡
Cursor feels like VS Code

Cursor is a fork of VS Code — it started from VS Code's source code and added a deeper AI layer on top. Your VS Code extensions, keyboard shortcuts, themes, and settings all work in Cursor. You can import them with one click when you first open Cursor. The transition cost is nearly zero.

Setup

Cursor installs like any application. If you already use VS Code, your settings follow you.

1

Download and install Cursor

Download from cursor.com. On first launch, Cursor will ask if you want to import your VS Code settings, extensions, and keybindings. Click yes. Your environment is immediately familiar.

2

Sign in and choose a plan

Cursor offers a free tier with limited AI requests per month. The Pro plan (~$20/month) gives higher limits and access to more powerful models. You can start on the free tier to evaluate it.

3

Choose your model

In Cursor's settings, you can select which AI model powers it. The main options:

Claude Sonnet (default/recommended) — strong reasoning and instruction-following, good balance of speed and quality
GPT-4o — fast, capable, good for quick tasks
Gemini 1.5 Pro — very large context window; reads more files simultaneously

Start with the default. You can switch models per task as you get more comfortable.

4

Open your existing project

Open Cursor → File → Open Folder. Open any project you were already working on in VS Code. Your virtual environment, Python interpreter selection, and Git setup carry over. Nothing about your project needs to change.

The four AI modes — and when to use each

Cursor has four distinct ways to interact with AI. Using the right one for the situation is the key skill at Level 2.

Tab

Inline autocomplete

The same as Level 1 — AI suggests the next line or block as you type. The difference in Cursor: suggestions are informed by your entire codebase, not just the open file. Variable names, patterns, and conventions from other files appear in your completions.

Use for: Day-to-day typing. Writing code that follows patterns you've already established elsewhere.
⌘K

Inline edit

Select code → press Cmd+K → describe the change. A diff appears showing the before and after. Accept with Cmd+Y, reject with Cmd+N.

This is the fastest way to make targeted changes to existing code without a conversation:

"Add type hints to all parameters and the return value"
"Rewrite this to handle the case where the list is empty"
"Split this into two functions — one for loading, one for cleaning"
Use for: Targeted rewrites of specific functions. Adding error handling, type hints, logging. Splitting large functions. Anything where you know what you want changed but writing it yourself is tedious.
⌘L

Chat with codebase context

Press Cmd+L to open the chat panel. Like Level 1 chat, but with a critical addition: the @ reference system lets you pull specific files, folders, or your entire codebase into context.

@codebase How is error handling currently done across the project?
@analysis.py @utils.py These two files need to share logic — what's the best way to refactor this?
@web Find the pandas documentation for GroupBy aggregations
Use for: Questions that span multiple files. Architecture and design decisions. Understanding how parts of your codebase relate. Anything where you need AI to have broad context before answering.
Agent

Agent mode — goal-directed execution

The defining Level 2 capability. You give a goal; Cursor plans the work, reads the relevant files, writes the changes across as many files as needed, runs the result if applicable, and shows you diffs to review.

"Write pytest tests for every function in src/ and make sure they all pass"
"Add logging to every function that does a file operation"
"Refactor the data loading module to use a class instead of standalone functions"
⚠
Always read the diffs before accepting

Agent mode can change many files at once. Cursor shows you a diff for every change. Read them. The agent is right most of the time — but “most of the time” is not “always,” and confident-looking code can still have subtle bugs. Your review is the quality gate.

Use for: Tasks that span multiple files. Generating a full test suite. Large refactors. Repetitive changes across a codebase. Anything that would take you 30 minutes to do manually.

A real Level 2 session

Here's how the five-step workflow looks when you're working at Level 2. The steps are the same — the way you use AI at each step is different.

Organize

Use chat to plan the structure before creating any files

Press Cmd+L. Describe what you're building. Ask Cursor to propose a file structure and module breakdown. Read the proposal critically — does it make sense for your project? Adjust before creating anything.

Write

Alternate between Tab completions and Cmd+K edits

Start files with Tab completions for routine code. When you need to modify something specific, select it and use Cmd+K. When a task would take you 10+ minutes, switch to Agent mode and give it a goal.

Test

Ask agent to write and run tests

Once your code works, open Agent mode: “Write pytest tests for everything in analysis.py and run them. Fix any failures you find.” Agent writes the tests, runs them, reads the failures, and iterates. You review the final test file.

Share

Git workflow is unchanged

Commit and push the same way as Level 1. Cursor has a Source Control panel (same as VS Code). One addition: before committing, you can ask Cursor to review the diff — “Review my staged changes. Is there anything I should fix before committing?”

Staying in control at Level 2

Level 2 gives AI more autonomy. These habits keep you in the driver's seat.

Read every diff before accepting

Cursor shows a diff for every change the agent proposes. Make reading these non-negotiable. A two-minute diff review now prevents an hour of debugging later. If you don't understand a change, ask Cursor to explain it in the chat panel before accepting.

Keep tasks focused

Agent mode works best on tasks with a clear scope. “Add tests for the data loading module” is good. “Improve the whole project” is not. The more specific the goal, the more reviewable the output.

Run the code after every agent session

Always run your test suite and key scripts after an agent session, before committing. Agent mode can make changes that look correct in a diff but break something unexpected. Tests catch this.

Use Git to protect yourself

Commit your working code before running agent mode on a large task. That way, if the agent's output isn't what you wanted, you can git diff to see exactly what changed and git checkout to undo it. Level 2 without Git is risky.

What's next — Level 3

Level 2 keeps AI inside the editor. Level 3 removes that constraint entirely. Claude Code runs in your terminal and has access to your full computing environment — it can read and write any file, run any script, install packages, and use the terminal exactly the way you do. The IDE becomes optional.

When is Level 3 the right next step? When you find yourself wanting to say “just do this whole task” without the overhead of an editor UI. When your tasks span the filesystem rather than just your project. When you want to understand how agents work without a visual layer on top.
