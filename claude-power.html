Level 3 — Dedicated Coding Agent

Claude Advanced

At Level 3, the IDE is optional. Claude Code runs in your terminal with full access to your environment — any file, any script, any tool. You describe what you want; Claude reads, writes, runs, and iterates. Seven integrations make it significantly more powerful.

What Level 3 adds beyond Level 2

Level 2 (Cursor)

AI works inside your editor. It sees your open files and your project folder. It can propose changes across multiple files. But it's still constrained by the editor environment.

Level 3 (Claude Code)

AI works in your terminal, with access to everything on your machine. It can run any script, install packages, use system tools, read and write anywhere on your filesystem, and execute workflows that span your entire computing environment.

What this changes in practice

At Level 2, you review diffs in an editor. At Level 3, you describe a task and come back to results. The five-step workflow is still there — but “Write” might mean “here's the spec; build it.” The IDE is still useful for reading the output, but it's no longer required to run the task.

How Claude Code works

Claude Code is a command-line tool. You install it once, navigate to your project, and type claude. A session starts. You talk to it in plain English. It reads your files, writes code, runs scripts, sees the output, and iterates — all in the terminal.

You type

Read my project and summarize what it does and how it's structured.

Claude does

Reads every file in the directory and gives a plain-English summary of the project's structure, purpose, and main components.

You type

Write a function in analysis.py that groups sales by region and returns a sorted DataFrame. Then write a test for it and run it.

Claude does

Writes the function in the correct file, writes a test, runs the test, reads the output, fixes any failure, and reports back with the result.

Installation

# Requires Node.js 18+ (download from nodejs.org if needed) npm install -g @anthropic-ai/claude-code # Navigate to your project and start a session cd my-project claude # Log in with your Anthropic account when prompted # The $20/month Pro plan includes Claude Code — no separate subscription
💡
Claude Code alongside your IDE

You don't have to choose between Claude Code and your IDE. Many developers keep VS Code or PyCharm open to read files and run the IDE's debugger, while running Claude Code in a terminal alongside it. Claude Code makes the changes; the IDE shows you what changed. They complement each other.

The core problem to understand: context

Before installing any integrations, you need to understand the single most important concept for working effectively with Claude Code: the context window.

When Claude Code starts a session, it knows nothing about your project. As you give it commands — read this file, write this function, run this test — all of that information accumulates in a “context window”: a fixed-size memory that gets passed back and forth with every message. Think of it like a notepad that Claude keeps adding to. The problem: once the notepad is full, Claude starts forgetting earlier entries to make room for new ones.

In practice, this means a Claude Code session can start sharp and become unreliable as context fills up — forgetting constraints you set earlier, writing duplicate code, losing track of decisions already made. This isn't a bug in Claude; it's a fundamental property of how these models work.

⚠
The rule: don't let context go past ~50%

Once the context window is around half full, Claude's quality degrades noticeably. The statusline (integration #1 below) shows you exactly what percentage is used. When it approaches 50%, end the session and start a fresh one. Before you do, ask Claude to write a summary of what it was working on to a file — the new session can start by reading that file.

# Before ending a long session: Write a summary of what we've done and what's left to do to progress.md # At the start of the next session: Read progress.md and continue from where we left off

The seven integrations below each address a different aspect of working effectively with Claude Code. The first — the statusline — makes context visible. The second — Superpowers — addresses it structurally by dispatching sub-agents with their own context windows. Everything else builds on that foundation.

01

Statusline

Make context usage visible — the most critical metric in a Claude Code session

By default, Claude Code gives you no live view of how much context you've used, what it's costing, or what model is active. The statusline adds a persistent bar at the bottom of your terminal session showing all of this in real time.

Why it matters

Context percentage is the most important number in a Claude Code session. Without it, you'll only notice context degradation when Claude starts making mistakes — by which point you've already wasted time on low-quality output. With the statusline, you can preemptively end sessions before quality drops.

What the statusline shows

Model name (e.g. claude-sonnet-4)
Context window percentage used — the critical number
Session cost in USD
Session elapsed time
Current Git branch and working tree status

Setup

Run this command in your terminal before starting a Claude Code session:

npx cc-statusline@latest

This installs and launches the statusline. You can configure which metrics appear in which order. A suggested Line 1 configuration: model · context % · session cost · elapsed time. Line 2: git branch · git worktree status.

⚠
On /compact: don't rely on it

Claude Code has a /compact command that summarizes the conversation and starts fresh. It's tempting but usually counterproductive — you lose some context, but the “poisoned” context from before the compaction still affects quality. It's better to end the session cleanly and start fresh with a written summary (as described in the context section above).

02

Superpowers

Structured sub-agent workflows — the most impactful single installation

Superpowers is a Claude Code plugin that fundamentally changes how Claude approaches tasks. Instead of one Claude session accumulating context until it degrades, Superpowers enables sub-agent workflows: your main Claude instance acts as an orchestrator, dispatching specialized sub-agents to do focused work — each in its own fresh context window — and reporting back results.

Why it matters

This solves the context window problem structurally. A sub-agent that writes code, a sub-agent that reviews it, a sub-agent that writes tests — each starts fresh. The orchestrator only receives the summaries, not the full workload. Superpowers also enforces a planning workflow that dramatically improves output quality: brainstorm first, write a spec, then implement from the spec.

The three core commands

superpowers brainstorm — start any project or feature. Claude asks clarifying questions, proposes 2–3 approaches, and produces a spec document for your approval. You review and steer.
superpowers write plan — takes your approved spec and turns it into a detailed implementation plan: file by file, function by function, with all the details needed for production-quality code.
superpowers execute plan — dispatches sub-agents to implement the plan. Each agent handles a focused piece of work with its own context window. Code reviewers and testers are automatically included.

Setup

# Inside a Claude Code session, type: /plugin # From the plugin menu, add and install superpowers: # Select "superpowers" and install for user scope # Also install "code-simplifier" while you're here # Exit and restart Claude Code — Superpowers is now active
💡
Install this first

If you install one thing from this page, install Superpowers. The shift from ad-hoc requests to plan-driven development with sub-agents is the biggest quality jump available in Claude Code. The brainstorm → plan → execute workflow is what serious teams use — Superpowers gives it to you automatically.

03

Sequential Thinking

Deeper reasoning for complex problems

Sequential Thinking is an MCP (Model Context Protocol) server that adds structured chain-of-thought reasoning to Claude Code. Before acting on complex tasks, Claude works through a visible step-by-step reasoning process: understand the problem → identify constraints → consider approaches → select and implement. This reasoning is visible to you and catchable when it goes wrong.

Why it matters

On straightforward tasks, Claude Code performs well by default. On complex architectural decisions, multi-step algorithms, or tasks with non-obvious constraints, forcing explicit reasoning before action produces significantly better results. Sequential Thinking combines especially well with Superpowers — the brainstorm and planning steps benefit most from deep reasoning.

What MCP means

MCP (Model Context Protocol) is an open standard that lets Claude Code connect to external tools and data sources as if they were built-in capabilities. An MCP server is a small program Claude Code can call. Sequential Thinking, Context7, and others are MCP servers. You add them to Claude Code's configuration once; they're available in every session.

Setup

The simplest install: ask Claude Code to do it for you:

# In a Claude Code session: Please install the sequential thinking MCP server

Claude Code will find the correct MCP package, install it, and configure it. Restart your Claude Code session when done. Sequential Thinking is then available in all future sessions.

04

Context7

Up-to-date library documentation — eliminates hallucinated APIs

Claude Code's training data has a cutoff date. When you ask it to use a library — pandas, FastAPI, SQLAlchemy — it works from what it learned during training, which may be months or years out of date. The result: Claude confidently uses deprecated methods, renamed functions, or APIs that no longer exist.

Context7 fixes this. It's an official Claude plugin that pulls current, version-specific documentation from live source repositories and injects it into Claude's context when you ask about a library. Claude answers based on the actual current state of the library, not outdated training data.

Setup

# Inside a Claude Code session: /plugin # Find Context7 in the plugin list (it's an official plugin) # Select it and install for everyone # Restart Claude Code — Context7 is now available

How to use it

Add use context7 to any prompt where you need current documentation:

Build a FastAPI endpoint that validates JSON input against a Pydantic model. use context7 How do I use pandas GroupBy with multiple aggregation functions in version 2.x? use context7

Context7 resolves the library name, fetches the current documentation for the relevant version, and injects it before Claude answers. No more “this method doesn't exist” bugs from outdated training data.

05

Warp

A better terminal for running Claude Code

Warp is a modern terminal application — a replacement for Mac's Terminal, Windows Terminal, or the terminal built into VS Code. You run Claude Code inside Warp. Warp's own built-in AI is not the point; what matters is Warp's features for managing Claude Code sessions.

The problem it solves

Running Claude Code in a standard terminal means your files and your Claude session are in separate windows. You work in Claude, switch to a file explorer to check what it wrote, switch back. Warp puts them side by side: Claude in one pane, your repo in another. When Claude writes a plan to docs/plan.md, you can click into it and read it while Claude is still working.

The features that matter for Claude Code

Split panes (Cmd+D)

Press Cmd+D to split your terminal into two panes. Run Claude Code on the left. Browse your repository files — or run another Claude Code instance — on the right. You can keep multiple Claude sessions running simultaneously for parallel tasks.

Tabs (Ctrl+T)

Create tabs with Ctrl+T. Each tab can have its own split panes. Use tabs to organize multiple projects or different types of terminal work (Claude Code in one tab, running your server in another).

File panel

Open the file panel to browse your repository inline. When Claude writes a spec or implementation plan to a file, you can read it without leaving the terminal. This is especially valuable when reviewing Superpowers plans before approving execution.

Block-based output

Every command and its output is a “block” — a discrete unit you can copy, share, or reference. No more scrolling through walls of output to find what Claude ran. Each action is visually distinct.

Setup

# Download from warp.dev (free to install) # Available for Mac, Linux, and Windows # Open Warp, navigate to your project, and run claude as usual cd my-project claude

Free tier: ~100 AI requests/month using Warp's own AI (which you don't need to use). Claude Code running inside Warp uses your Anthropic account separately from Warp's own AI.

06

Mobile — Happy Engineering

Control a real terminal session from your phone

The official Claude mobile app can't run local files or access your terminal. It has roughly 10% of Claude Code's capabilities. The real mobile solution is Happy Engineering — an open-source tool that runs a live terminal session on your computer that you control from your phone.

What this actually is

Happy Engineering creates a terminal on your computer that's accessible from a mobile browser. When you connect from your phone, you're controlling the actual terminal on your machine — not a sandboxed version. This means Superpowers, Context7, Sequential Thinking — everything you've configured — is available from your phone. Tasks you start on your phone continue running on your computer. You come home, pick up where you left off.

Setup

# Visit happy.engineering to see installation instructions # Copy the install command, run it in your terminal # Complete the brief phone setup # Open your computer's terminal session from your phone's browser

Free and open source. Requires your computer to be on and connected. Available for iOS, Android, and web. The computer stays running; you pick up the session when you return.

07

Skills

Custom reusable workflows for tasks you do repeatedly

Skills are reusable instruction sets stored as markdown files in your project. When you invoke a skill, Claude Code loads those instructions and follows them for the current task. Instead of re-explaining the same process every session, you encode it once and invoke it with a single command.

Why it matters

Every project has repetitive tasks: adding a new data source, creating a new model, running a specific analysis pipeline, generating a weekly report. Without skills, you re-explain the process each time or copy-paste from previous sessions. With skills, one command runs the full workflow with all your project's conventions already baked in. As you build up a library of skills, your Claude Code sessions become dramatically more efficient.

How skills work

# Skills are markdown files in .claude/skills/ .claude/ skills/ add-data-source.md ← invoked with /add-data-source run-analysis.md ← invoked with /run-analysis review.md ← invoked with /review

Building a skill with Superpowers

Rather than writing skills from scratch, Superpowers can build them for you. Describe the repetitive task in detail — what it involves, what files it touches, what decisions it requires, what the output looks like — and ask Superpowers to create a skill:

# In a Claude Code session with Superpowers installed: I want to create a skill for adding a new analysis module to this project. Every new module needs: a Python file in src/, a test file in tests/, an import added to __init__.py, and a section in README.md. Build me a skill called "add-module" that handles all of this.

Superpowers uses Sequential Thinking to build the skill carefully. On the first run, give feedback on what's not quite right. After a few iterations, the skill becomes a reliable automation you can use every session.

Using skills

# Invoke any skill with its name: /add-module data_cleaning /run-analysis quarterly_report /review src/analysis.py
