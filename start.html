Hands-on exercise

Your first project — start to finish

Git · GitHub · venv · pytest · the whole workflow. Opens with Windows + X → I to launch PowerShell.

prereqs

Before you start, make sure you have:

Git — git-scm.com
GitHub CLI — cli.github.com
A GitHub account — github.com

VS Code and Python are already installed.

step 1

Organize — Create your folder and open VS Code

Press Windows + X then I to open PowerShell. Type these one at a time:

cd Desktop  ← navigate to your Desktop
mkdir tip_calculator  ← create your project folder
cd tip_calculator  ← move into it
code .  ← open VS Code here

Switch to the terminal inside VS Code with Ctrl + ` — use this for everything from here on.

step 2

Organize — Git, virtual environment, and files

Tell Git who you are — one-time setup, never needs repeating:

git config --global user.email "you@example.com"  ← your GitHub email
git config --global user.name "Your Name"  ← your name

Then set up your project:

git init  ← create a local git repository in this folder
python -m venv .venv  ← create a virtual environment called .venv
.venv\Scripts\activate  ← activate it — your prompt will show (.venv)

Create your project files:

New-Item tip_calculator.py  ← your main code file
New-Item test_tip_calculator.py  ← your test file
New-Item .gitignore  ← tells Git what folders to skip

Open .gitignore in VS Code, paste the following in, and save with Ctrl + S:

.venv/  ← your virtual environment — hundreds of files, not needed on GitHub
__pycache__/  ← Python's temp files, auto-created when you run code
.pytest_cache/  ← pytest's temp files, auto-created when you run tests

None of these belong on GitHub. They are local, auto-generated, and irrelevant to anyone else.

step 3

Organize — Install pytest before writing anything

Do this before writing any code so VS Code doesn't show errors as you type:

pip install pytest  ← install pytest into your .venv
pip freeze > requirements.txt  ← save your package list

requirements.txt is how you tell anyone who clones your project which packages to install. It replaces the need to share your .venv folder.

step 4

Write — Add your code and tests

pytest is a package that checks whether your code does what you expect. You write a separate test file containing functions that call your real code with specific inputs and assert what the output should be. When you run pytest in the terminal, it finds those functions automatically and runs them all. A dot means a test passed. An F means it failed, and pytest tells you exactly what went wrong.

Open tip_calculator.py and paste this in, then save:

def calculate_tip(bill_amount, tip_percent):
    """Calculate the tip amount for a bill."""
    if bill_amount < 0:
        raise ValueError("Bill amount cannot be negative")
    return round(bill_amount * (tip_percent / 100), 2)

Open test_tip_calculator.py and paste this in, then save:

import pytest
from tip_calculator import calculate_tip
 
def test_standard_tip():
    assert calculate_tip(100, 20) == 20.0
 
def test_zero_tip():
    assert calculate_tip(50, 0) == 0.0
 
def test_negative_bill_raises_error():
    with pytest.raises(ValueError):
        calculate_tip(-10, 20)

step 5

Test — Run pytest

pytest  ← runs all test files it finds automatically

You should see:

collected 3 items
test_tip_calculator.py ...
3 passed in 0.12s

After this runs, two new folders will appear in VS Code's file explorer — __pycache__/ and .pytest_cache/. pytest created these automatically. You don't need to touch them — your .gitignore already tells Git to ignore them.

step 6

Share — Push to GitHub

Log in to GitHub from the terminal — one-time setup:

gh auth login  ← follow the prompts, it will open a browser to confirm

Then create your repo and push your code in one command:

gh repo create tip_calculator --private --source=. --remote=origin --push  ← creates the repo, links it, and pushes your code

Go to github.com/yourusername/tip_calculator — your files are there.

loop

The core loop going forward

edit code  ← make changes in VS Code
pytest  ← confirm everything still works
git add .  ← stage all changes
git commit -m "describe what you changed"  ← save a snapshot
git push  ← send it to GitHub

Every git commit is a saved snapshot. Every git push sends it to GitHub.

VS Code — Flow 1 of 2

Setup & First Run

Install Python, VS Code, create your first project, set up a virtual environment, add AI, and push to GitHub.

1

Install Python

Download from python.org. On Windows, check “Add Python to PATH” during installation — this is critical. On Mac, Python may already be installed, but download the latest from python.org for a clean setup.

python --version
# Should show: Python 3.12.x or similar
2

Install VS Code and the Python extension

Download from code.visualstudio.com. Once open, click the Extensions icon in the left sidebar (four squares) and search for “Python” — install the one by Microsoft. This gives VS Code Python understanding: syntax highlighting, autocomplete, environment detection, and a debugger.

VS Code is folder-based. You always open a project folder, not individual files.
3

Create a project folder and virtual environment

Go to File → Open Folder. Create a new folder (e.g. my-project) and open it. Then open the built-in terminal (Terminal → New Terminal) and run:

# Create the virtual environment
python -m venv .venv

# Activate it (Mac/Linux)
source .venv/bin/activate

# Activate it (Windows)
.venv\Scripts\activate

When activation succeeds, your prompt shows (.venv) at the start. VS Code may prompt you to select this as the interpreter — click Yes.

4

Create your project structure and install packages

Create a simple structure. In the terminal:

# Install a package (with your venv active)
pip install pandas requests

# Save dependencies so others can recreate your environment
pip freeze > requirements.txt

In VS Code's file explorer, create a file called main.py. This is your entry point. Also create a .gitignore file now — paste .venv/, __pycache__/, *.pyc, .env into it.

Always activate your virtual environment before installing packages. If (.venv) isn't in your prompt, packages go into your global Python instead.
5

Add AI: install GitHub Copilot

In the Extensions panel, search for “GitHub Copilot” and install it. Also install “GitHub Copilot Chat.” Sign in with a GitHub account. Copilot is $10/month — free for verified students at education.github.com.

Alternatively, search for “Gemini Code Assist” (free with a Google account) or “Codeium” (permanently free for individuals).

Once installed, you'll see gray inline suggestions as you type. Press Tab to accept, keep typing to ignore.

You can use multiple AI tools simultaneously. Many developers run Copilot for inline completions and open claude.ai in a browser tab for deeper questions.
6

Run your first script

In main.py, type a simple script:

import pandas as pd

data = {'name': ['Alice', 'Bob'], 'score': [92, 85]}
df = pd.DataFrame(data)
print(df)

Press F5 to run (or right-click → Run Python File). You should see the DataFrame printed in the terminal below. If Python can't find pandas, check that your .venv interpreter is selected — press Cmd/Ctrl+Shift+P → “Python: Select Interpreter” → choose the .venv entry.

7

Set up Git and push to GitHub

See the Git & GitHub setup section below — the steps are the same regardless of which IDE you use.

VS Code — Flow 2 of 2

Working with AI at Level 1

A real project walkthrough — planning, writing, testing, debugging, and refactoring with AI alongside you at every step.

This walkthrough uses a small data analysis project as the running example: loading a CSV file, cleaning it, and computing summary statistics. The AI techniques apply to any Python project.

1

Plan with AI before writing a line

Before you open a new file, open the Copilot Chat panel (Ctrl+Shift+I) or claude.ai in a browser. Describe what you're building:

# Example prompt
I want to write a Python script that: 1. Loads a CSV file with columns: date, product, region, sales 2. Cleans the data (handle missing values, parse dates) 3. Computes total sales by region and by month 4. Prints a summary table How should I structure this? What functions should I create?

Read the response carefully. You don't have to follow it exactly — but starting with a plan prevents the most common mistake: writing one big function that does everything.

This is one of AI's most underused capabilities. Students often go straight to writing code. Planning first with AI gets you a better structure in 5 minutes than you'd arrive at by writing for an hour.
2

Use comments to drive inline generation

In your Python file, write a comment describing what you want, then press Enter. Copilot reads the comment and suggests the implementation below it:

# Load CSV file and parse the date column as datetime
# Copilot suggests the function below — press Tab to accept:
def load_data(filepath):
    df = pd.read_csv(filepath, parse_dates=['date'])
    return df

Read what it suggests before accepting. Is it correct? Does it match your intent? Press Tab to accept. If the suggestion isn't right, keep typing your own version — the suggestion disappears.

You're in control of scope. A focused, specific comment gets you a focused function. "Do everything" gets you something overwhelming. One comment, one function.
3

Ask the chat panel when you're stuck

Hit a wall? Select the problematic code, then open Copilot Chat. Ask specifically:

# Good: specific, includes the context
"This function should group sales by month, but it's returning an empty DataFrame. Here's the code. What's wrong?"

# Less useful: vague
"Why isn't this working?"

The more specific your question and the more context you give (paste the code, paste the error), the more useful the answer. AI isn't magic — it responds to the quality of what you give it.

4

Debug errors with AI: read the traceback, then ask

Run your script. If you get an error, read the traceback before pasting it to AI. Try to locate which line caused it. Then:

# In the Copilot Chat panel:
"I got this error running my data cleaning function: KeyError: 'sales' The traceback shows it's on line 14. Here's the full function. What does this mean and how do I fix it?"

AI will explain what a KeyError means (a dictionary key or DataFrame column doesn't exist), suggest why it's happening (the column might be named differently in the CSV), and show you how to fix it.

Reading the error yourself first builds the skill of understanding Python errors. Use AI to confirm and go deeper — not as the first line of defense.
5

Get code explained to you

Select any code you don't fully understand — something Copilot generated, a library function, or something you wrote last week. In the Copilot Chat panel:

"Explain this function line by line. What does each part do and why is it written this way?"

This is how you learn from AI rather than just depending on it. Every function you accept without understanding is technical debt — at some point you won't be able to maintain it. Use the explanation request liberally.

6

Generate tests with AI

Once a function is working, select it and ask Copilot Chat to write tests:

"Write pytest tests for this function. Test: - The normal case with valid data - What happens with an empty DataFrame - What happens if the date column is missing"

Create a tests/ folder and a file like test_analysis.py. Paste the generated tests, review them, and run them:

pip install pytest
pytest tests/
AI-generated tests are a starting point, not a complete suite. They cover obvious cases — you should add tests for anything the AI missed that you know matters.
7

Refactor working code

Once your code works, AI can help you improve it. Select a function and ask:

"Review this function. Is there a more Pythonic way to write it? Are there any edge cases I'm not handling? Keep the behavior the same."

The AI will suggest improvements: more idiomatic Python, better variable names, extracted helper functions, or missing error handling. You decide what to accept. Refactoring with AI is a great way to improve code quality and learn what good Python looks like.

8

Add documentation

Select a function and ask for a docstring:

"Write a Google-style docstring for this function, including parameter types, return value, and an example."

For the whole project, ask for a README:

"Write a README.md for this project. Include: what it does, how to install dependencies, and how to run it."

Documentation is the thing developers skip most often. With AI it takes two minutes.

💡
The principle behind all of this

AI works best when you stay specific and stay in control. One focused prompt gets you one useful output. Vague prompts get vague code. Read everything AI generates before accepting it. When you don't understand something, ask AI to explain it — that's where the learning happens.

PyCharm — Flow 1 of 2

Setup & First Run

Install PyCharm, create a project (virtual environment is automatic), add AI, and run your first script.

1

Download and install PyCharm

Go to jetbrains.com/pycharm. Download Community (free) or Professional. Students get Professional free at jetbrains.com/student — sign up with a .edu email. The student license includes JetBrains AI, the full debugger, and database tools.

Community covers all core Python development. Professional adds: web frameworks, database tools, Jupyter notebooks, and JetBrains AI.

2

Create a new project

Open PyCharm → New Project. Choose a location. PyCharm will show a “Python Interpreter” section — select “New environment using Virtualenv.” PyCharm creates and manages the .venv folder automatically. You never activate or deactivate it manually — PyCharm handles this.

PyCharm's biggest advantage over VS Code: virtual environment management is completely handled for you, with a UI for installing packages without touching the terminal.
3

Install packages

Two options:

Via GUI: Settings → Project → Python Interpreter → the “+” button. Search for any package and install with one click.

Via terminal: PyCharm's built-in terminal (bottom of the screen) automatically has the virtual environment active. Type:

pip install pandas requests
pip freeze > requirements.txt

Also create a .gitignore file in the project root with: .venv/, __pycache__/, *.pyc, .env.

4

Add AI: install GitHub Copilot or enable JetBrains AI

JetBrains AI (Pro/student license): Already available in the right sidebar — look for the AI chat icon. No additional installation. JetBrains AI has access to your run logs, test results, and project structure in a way external tools can't match.

GitHub Copilot (Community edition): Settings → Plugins → Marketplace → search “GitHub Copilot” → Install. Sign in with your GitHub account. Works the same as in VS Code: inline gray suggestions, Tab to accept.

Codeium (free): Same plugin marketplace, search “Codeium.” Free for individuals, no subscription required.

5

Run your first script

Create a file main.py and type:

import pandas as pd

data = {'name': ['Alice', 'Bob'], 'score': [92, 85]}
df = pd.DataFrame(data)
print(df)

Press Shift+F10 to run (or the green triangle in the top-right toolbar). Output appears in the Run panel at the bottom. If PyCharm shows an import error, it will offer to install the missing package inline — click the suggestion.

6

Set up Git and push to GitHub

See the Git & GitHub setup section below. PyCharm also has a built-in Git interface: VCS → Enable Version Control Integration. You can commit, push, and view history without the terminal if you prefer.

PyCharm — Flow 2 of 2

Working with AI at Level 1

PyCharm's AI is more tightly woven into the IDE than VS Code's. Here's how to use it throughout a real project.

PyCharm surfaces most AI actions through right-click menus and the AI Chat panel. The keyboard shortcut Alt+Enter opens quick-fix suggestions — often the fastest path to AI help on a specific line.

1

Plan with the AI Chat panel

Open the AI Chat panel (right sidebar or View → Tool Windows → AI Assistant). Describe what you're building — PyCharm's AI can already see your project structure, so you don't need to re-explain it:

"I'm building a data analysis script for sales data. The CSV has columns: date, product, region, sales. How should I structure the functions? What should each one do?"

Ask for a plan, read it critically, and start from there. Don't accept a plan you don't understand.

2

Use AI completions while you type

Start typing a function. As you write, Copilot or JetBrains AI suggests completions inline. PyCharm's native type inference also kicks in — it knows what methods are available on a pandas DataFrame, what arguments a function expects, and what types you're working with. The AI and PyCharm's own intelligence work together.

Hold Alt to cycle through multiple AI suggestions if more than one is offered.

3

Use AI Actions from the right-click menu

Select any code → right-click → AI Actions. You'll see options including:

  • Explain code — plain-English explanation of the selected block
  • Suggest refactoring — improvements to structure and readability
  • Add error handling — wraps code in try/except with appropriate handling
  • Generate docstring — writes documentation for the function
  • Find potential bugs — scans for common mistakes

Every action shows a diff before applying. Read the diff. Accept what makes sense, reject the rest.

PyCharm's right-click AI menu is the fastest way to act on a specific piece of code. You don't need to write a prompt — the menu already knows the most useful actions.
4

Debug with the debugger + AI explanation

Click in the gutter next to any line to set a breakpoint (red dot). Press Shift+F9 to run in Debug mode. Execution pauses at the breakpoint. You can inspect every variable in the Debug panel.

When you hit an exception, the Debug panel shows the exact state of the program when it crashed. Then open AI Chat and ask:

"My script crashed with a KeyError on line 22. Looking at the debug panel, the DataFrame has columns ['date', 'product', 'Revenue'] but my code references 'sales'. How should I fix this and prevent it in the future?"

The combination of PyCharm's debugger (exact program state) and AI (explanation and fix) is the most powerful debugging setup available at Level 1.

5

Generate tests with AI

Right-click on a function → Generate → Tests. PyCharm opens a dialog, creates the test file in the correct location, and — with JetBrains AI — pre-populates test cases. Review them:

# Review generated test cases like this one:
def test_load_data_missing_column():
    # What happens if 'sales' column is missing?
    # AI should have generated this edge case

Run tests with a single click from the test file (green arrow) or from the Run panel. Red = failing, green = passing. Ask AI in the chat panel to explain any failing test.

6

Rename and refactor safely

PyCharm's refactoring tools are Python-aware, not just text search-and-replace. Right-click any variable or function name → Refactor → Rename. PyCharm renames it everywhere it's used across the entire project, understanding scope correctly.

For larger refactors, use AI Chat: “This function is doing too many things. How should I split it up?” Let AI propose the structure, then use PyCharm's Extract Method (right-click → Refactor → Extract → Method) to do the splitting safely.

7

Use AI for documentation

Place your cursor inside any function → right-click → AI Actions → Generate Docstring. PyCharm writes the docstring based on the function's actual parameters and body. For the project README, open AI Chat:

"Write a README.md for this project. You can see the files and structure — include: what it does, how to install, and how to run it."

JetBrains AI can see your project structure directly, so the README it generates is specific to your actual project, not a generic template.

💡
PyCharm + AI: what makes it different

In VS Code, AI is an extension sitting on top of the editor. In PyCharm, the AI and the IDE share a complete picture of your project — your files, your run history, your test results. JetBrains AI can answer “why did my last test fail?” because it saw the test run. That shared context makes the AI answers more accurate for Python-specific questions.

Universal — same for all IDEs

Git & GitHub Setup

Version control setup is identical regardless of which IDE you're using.

1

Install Git and configure your identity

Download from git-scm.com. On Mac, Git may already be installed — type git --version to check. On Windows, the installer includes Git Bash.

# One-time setup — your name and email appear in commit history
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
2

Create a GitHub account

Go to github.com and create a free account. Your username is public and often serves as a professional identifier.

3

Initialize Git in your project

In your project folder's terminal:

git init
git add .
git commit -m "Initial commit"

Make sure your .gitignore is in place before running git add . so your .venv folder isn't included.

4

Create a repository on GitHub and push

On GitHub, click “New repository.” Give it a name. Don't initialize with a README (you already have code locally). Copy the commands GitHub gives you:

git remote add origin https://github.com/yourusername/your-repo.git
git branch -M main
git push -u origin main

Your code is now on GitHub. Every future commit pushes with just git push.

5

The daily Git rhythm

# See what's changed
git status

# Stage all changes
git add .

# Commit with a clear message
git commit -m "Add data cleaning function"

# Push to GitHub
git push
Both VS Code and PyCharm have visual Git interfaces. In VS Code: the Source Control panel (left sidebar). In PyCharm: VCS menu and the Commit panel. You don't have to use the terminal for Git once you're set up.

Common pitfalls — and how to avoid them

These are the mistakes almost everyone makes at least once. Knowing them now saves hours of confusion.

⚠

Installing packages without activating your virtual environment

If your terminal prompt doesn't show (.venv), the environment isn't active. Packages installed now go into your global Python instead of your project's environment.

Fix: source .venv/bin/activate (Mac/Linux) or .venv\Scripts\activate (Windows)
⚠

Committing your .venv folder to Git

The virtual environment is large, machine-specific, and fully reproducible from requirements.txt. It should never be in your repository.

Fix: Add .venv/ to .gitignore before your first commit.
⚠

Committing API keys or secrets

Even a brief push of a file containing a key is a security risk. GitHub scans for key patterns. If it happens, assume the key is compromised and rotate it immediately.

Fix: Store secrets in a .env file and add it to .gitignore. Load with python-dotenv.
⚠

VS Code pointing at the wrong Python interpreter

VS Code may have multiple Python installations to choose from. If a package you installed can't be found, your IDE is probably pointing at global Python instead of your .venv.

Fix: Cmd/Ctrl+Shift+P → “Python: Select Interpreter” → choose the one inside .venv.
⚠

Accepting AI code without reading it

AI generates plausible-looking code that can have subtle bugs. Every piece of AI-generated code you accept without reading is a bug you've agreed to be responsible for.

Fix: Read every suggestion before pressing Tab or Apply. When you don't understand a line, ask AI to explain it.
⚠

Forgetting to update requirements.txt

Installing a new package doesn't automatically update requirements.txt. If a teammate clones your repo and installs from it, they won't have your new package.

Fix: Run pip freeze > requirements.txt after installing any new package, and commit it.
