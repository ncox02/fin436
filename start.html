Get started

The whole workflow. Do this first.

First Steps What's Next
Platform:
Hands-on exercise · ~5 minutes
prereqs

Make sure you have:

Python — python.org
pip — included with Python; on some systems use pip3 and python3 instead of pip and python
Git — git-scm.com
GitHub CLI — cli.github.com
A GitHub account — github.com

VS Code and Python are already installed if you followed the setup guides.

step 1

Organize — Folder, Git, virtual environment, and files

Press Windows + X then I to open PowerShell. Type these one at a time:

Open Terminal from Spotlight (Cmd + Space, type “Terminal”). Type these one at a time:

bash
PS C:\Users\Student> ~ % cd Desktop  ← navigate to your Desktop
PS C:\Users\Student\Desktop> ~/Desktop % mkdir tip_calculator  ← create your project folder
PS C:\Users\Student\Desktop> ~/Desktop % cd tip_calculator  ← move into it
PS C:\Users\Student\Desktop\tip_calculator> ~/Desktop/tip_calculator % code .  ← open VS Code here

Switch to the terminal inside VS Code with Ctrl + ` — use this for everything from here on.

Tell Git who you are — one-time setup, never needs repeating:

bash
PS C:\Users\Student\Desktop\tip_calculator> ~/Desktop/tip_calculator % git config --global user.email "you@example.com"  ← your GitHub email
PS C:\Users\Student\Desktop\tip_calculator> ~/Desktop/tip_calculator % git config --global user.name "Your Name"  ← your name

Then set up your project:

bash
PS C:\Users\Student\Desktop\tip_calculator> ~/Desktop/tip_calculator % git init  ← create a local git repository in this folder
PS C:\Users\Student\Desktop\tip_calculator> ~/Desktop/tip_calculator % python -m venv .venv  ← create a virtual environment called .venv
PS C:\Users\Student\Desktop\tip_calculator> .venv\Scripts\activate  ← activate it — your prompt will show (.venv)~/Desktop/tip_calculator % source .venv/bin/activate  ← activate it — your prompt will show (.venv)

Create your project files:

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> New-Item tip_calculator.py  ← your main code file
(.venv) PS C:\Users\Student\Desktop\tip_calculator> New-Item test_tip_calculator.py  ← your test file
(.venv) PS C:\Users\Student\Desktop\tip_calculator> New-Item .gitignore  ← tells Git what folders to skip
bash
(.venv) ~/Desktop/tip_calculator % touch tip_calculator.py  ← your main code file
(.venv) ~/Desktop/tip_calculator % touch test_tip_calculator.py  ← your test file
(.venv) ~/Desktop/tip_calculator % touch .gitignore  ← tells Git what folders to skip

Open .gitignore in VS Code, paste the following in, and save with Ctrl + SCmd + S:

text
.venv/  ← your virtual environment — hundreds of files, not needed on GitHub
__pycache__/  ← Python’s temp files, auto-created when you run code
.pytest_cache/  ← pytest’s temp files, auto-created when you run tests

None of these belong on GitHub. They are local, auto-generated, and irrelevant to anyone else.

step 2

Install packages

Do this before writing any code so VS Code doesn’t show errors as you type:

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % pip install pytest  ← install pytest into your .venv
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % pip freeze > requirements.txt  ← save your package list

requirements.txt is how you tell anyone who clones your project which packages to install. It replaces the need to share your .venv folder.

Having trouble? Ask AI.

Some commands may not execute as expected. Software changes, packages get updated, and platform differences can cause issues. For example, Python 3.14 introduced changes that affect how virtual environments (.venv) behave on some systems.

If something isn't working, paste the error message into Claude or ChatGPT and describe what you were trying to do.

step 3

Write — Add your code and tests

pytest is a package that checks whether your code does what you expect. You write a separate test file containing functions that call your real code with specific inputs and assert what the output should be. When you run pytest in the terminal, it finds those functions automatically and runs them all. A dot means a test passed. An F means it failed, and pytest tells you exactly what went wrong.

Open tip_calculator.py and paste this in, then save:

python
def calculate_tip(bill_amount, tip_percent):
    """Calculate the tip amount for a bill."""
    if bill_amount < 0:
        raise ValueError("Bill amount cannot be negative")
    return round(bill_amount * (tip_percent / 100), 2)

Open test_tip_calculator.py and paste this in, then save:

python
import pytest
from tip_calculator import calculate_tip

def test_standard_tip():
    assert calculate_tip(100, 20) == 20.0

def test_zero_tip():
    assert calculate_tip(50, 0) == 0.0

def test_negative_bill_raises_error():
    with pytest.raises(ValueError):
        calculate_tip(-10, 20)
step 4

Test — Run pytest

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % pytest  ← runs all test files it finds automatically

You should see:

output
collected 3 items
test_tip_calculator.py ...
3 passed in 0.12s

After this runs, two new folders will appear in VS Code’s file explorer — __pycache__/ and .pytest_cache/. pytest created these automatically. You don’t need to touch them — your .gitignore already tells Git to ignore them.

step 5

Share — Push to GitHub

Log in to GitHub from the terminal — one-time setup:

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % gh auth login  ← follow the prompts, it will open a browser to confirm

Then create your repo and push your code in one command:

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % gh repo create tip_calculator --private --source=. --remote=origin --push  ← creates the repo, links it, and pushes your code

Go to github.com/yourusername/tip_calculator — your files are there.

loop

The core loop going forward

bash
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % edit code  ← make changes in VS Code
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % pytest  ← confirm everything still works
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % git add .  ← stage all changes
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % git commit -m "describe what you changed"  ← save a snapshot
(.venv) PS C:\Users\Student\Desktop\tip_calculator> (.venv) ~/Desktop/tip_calculator % git push  ← send it to GitHub

Every git commit is a saved snapshot. Every git push sends it to GitHub.

What you just did

You completed the entire software development lifecycle: from an empty folder to a working project on GitHub with automated tests, version history, and a repeatable workflow.

Your .venv isolates dependencies. Your pytest suite ensures code correctness. Your Git history is a record you can revert to. Your GitHub repo is your backup and collaboration mechanism.

The loop you now run (edit → test → commit → push) is the same loop used to build production systems at scale.

Everything that comes next in Levels 2, 3, and 4 builds on this foundation. The next levels show you how to move faster within it.
